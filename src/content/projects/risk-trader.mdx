## Overview

A trading operating system for discretionary traders that connects to exchanges via API and enforces structured risk management, execution discipline, and behavioral review. The focus is on reducing emotional trading mistakes, enforcing predefined rules, and helping traders refine their process over time — the kind of tooling normally only available in-house at prop firms.

## Design Goals

- **Professional UI** — dense, fast, and information-rich, closer to a trading terminal or mission control system than a typical dashboard
- **Realtime architecture** — WebSocket event streams, event-driven state, replayable timelines
- **Modular rule system** — rules can be enabled/disabled with custom thresholds, built for future plugin-style expansion
- **Multi-exchange architecture** — exchange integrations sit behind a unified adapter interface
- **Paper trading mode** — simulated trading without real risk, still specific to the exchange being simulated

## Security: Zero-Knowledge API Keys

One of the more interesting technical decisions: the backend can never decrypt your exchange API keys. When you authenticate with a passkey, the browser derives a deterministic AES-256-GCM key from the WebAuthn PRF (Pseudo-Random Function) extension — a key that never leaves the browser. Only the encrypted ciphertext is stored on the backend, keyed by user and exchange, so a full backend breach exposes no usable keys. Decryption happens client-side via the WebCrypto API, with keys held only in a short-lived in-memory object rather than a reactive store, and wiped after use. Browsers without PRF support are blocked from storing API keys at all rather than falling back to something weaker.

## Architecture

- **Backend**: Node.js + TypeScript on Fastify, with Supabase for auth and data storage, Zod for validation
- **Frontend**: Vue 3 + TypeScript + Pinia, split into three layers — a Data layer (API/WebSocket calls normalized into ViewModels), a Service layer (singletons orchestrating higher-level operations), and a Presentation layer (smart components wired to services, feeding reusable dumb components)
- **Exchange adapters** abstracted behind a unified `ExchangeAdapter` interface, so new exchanges can be added without touching the rest of the system
- Passkey-only authentication (WebAuthn, no passwords), PostHog for analytics, GitLab CI/CD, Docker for deployment

## What's Built So Far

There's real code behind this already: a full Terminal view (positions, order entry, order book, chart panel, connection switcher, rule violation warnings), plus Journal, Analytics, Settings, Onboarding, and Login views. The backend has routes for rules, the trading journal, exchange credentials, passkeys, chart drawings, and settings, and the shared package defines the `ExchangeAdapter` interface every exchange integration will implement.

What's still missing: an actual exchange adapter (Binance, Bybit) wired up end-to-end — so nothing can place a real trade yet. That's the next milestone.

## Future Expansion

Mobile companion app, prop-firm mode, team/shared accounts, AI-assisted trade review, strategy statistics, market condition classification, custom scripting/rules, a desktop app, voice alerts, performance benchmarking, replay sharing, and more advanced telemetry dashboards.
